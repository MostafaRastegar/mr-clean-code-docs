# Clean Code Report Template

This directory contains a beautiful, interactive HTML report template for displaying clean code violation reports generated by the Cline rules system.

## Overview

The `clean-code-report.html` template provides a professional, user-friendly interface for viewing and analyzing clean code violations. It includes:

- **Summary Dashboard**: Overview of violation counts by severity
- **Interactive Filtering**: Filter violations by severity level
- **Search Functionality**: Search violations by message, code, or category
- **Detailed Modal View**: Click on any violation for detailed information
- **Responsive Design**: Works on desktop, tablet, and mobile devices

## Usage

### Basic Usage

1. **Generate a JSON report** using the Cline rules system
2. **Copy the HTML template** to your reports directory
3. **Update the JavaScript** to load your JSON data
4. **Open the HTML file** in any modern browser

### Integration Example

```html
<!-- Replace the sample data in the script tag -->
<script>
    // Load your JSON report data
    const reportData = {
        "report": {
            "metadata": {
                "version": "1.0.0",
                "timestamp": "2024-01-01T00:00:00Z",
                "file": "path/to/your/file.js",
                "totalViolations": 25,
                "severitySummary": {
                    "critical": 0,
                    "high": 15,
                    "medium": 8,
                    "low": 2
                }
            },
            "violations": [
                // Your violation objects here
            ]
        }
    };
</script>
```

### Loading External JSON

For dynamic reports, you can load JSON data from an external file:

```javascript
// Replace the static data loading with an AJAX call
async function loadReportData() {
    try {
        const response = await fetch('path/to/your/report.json');
        const reportData = await response.json();
        init(reportData);
    } catch (error) {
        console.error('Failed to load report data:', error);
    }
}

// Call this instead of init()
loadReportData();
```

## Customization

### Colors & Styling

```css

    --primary-color: #2563eb;      /* Main brand color */
    --critical-color: #dc2626;     /* Critical severity */
```

## Troubleshooting

### Common Issues

1. **JSON Data Not Loading**
   - Check that your JSON file is valid
   - Verify the file path is correct
   - Ensure CORS is not blocking the request

2. **Styling Issues**
   - Check browser console for CSS errors
   - Verify all CSS is properly loaded
   - Test in different browsers

3. **JavaScript Errors**
   - Open browser developer tools
   - Check the console for error messages
   - Verify all variables are properly defined

### Getting Help

If you encounter issues:

1. Check the browser's developer console for errors
2. Verify your JSON data structure matches the expected format
3. Test with the provided example data first
4. Review the CSS and JavaScript for syntax errors
