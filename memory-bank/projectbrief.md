# Project Brief

## Project Overview
Clean Code Rules for JavaScript/TypeScript is a comprehensive documentation repository containing clean code guidelines specifically tailored for JavaScript and TypeScript development.

## Core Requirements
- Provide comprehensive clean code guidelines for JavaScript/TypeScript
- Organize guidelines into logical, actionable categories
- Include practical examples with before/after code
- Support both frontend and backend development contexts
- Be suitable for code reviews, team onboarding, and personal reference

## Project Goals
1. **Educational Resource**: Serve as a learning tool for developers at all levels
2. **Code Quality Standard**: Establish consistent coding standards across teams
3. **Reference Material**: Provide quick access to best practices and patterns
4. **Community Contribution**: Allow for community input and improvements

## Scope
- JavaScript and TypeScript specific guidelines
- Covers naming conventions, function design, class design, error handling
- Includes testing, code organization, performance, security, and documentation
- Practical examples and real-world scenarios
- Common pitfalls and how to avoid them

## Success Criteria
- Comprehensive coverage of clean code principles for JS/TS
- Clear, actionable guidelines with practical examples
- Well-organized structure for easy navigation
- Community contributions and feedback integration
- Regular updates to reflect evolving best practices
